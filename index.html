The EOS Aurora Agents — Agentforce support across all 4 programme phases

From qualifying a market to trusting its deployment, Agentforce agents support Salesforce consultants and client stakeholders at every phase — not necessarily one agent per phase, but a continuous, orchestrated Agentforce layer — with a human checkpoint at every critical handoff.

Executive summary
4 phases, the key milestones, who does what

Agentforce agents support the programme end to end — the exact split (one continuous agent, several specialized agents, or shared Topics across phases) is still to be confirmed with the solution/architecture team. At every phase, the SF consultant frames and validates, the agent executes in direct contact with the client stakeholder — with a human checkpoint before handing the output to the next phase.

Detailed overview
Agentforce agents, four phases, one continuous thread

Each phase has its own operating model — but all of them follow the same logic: the SF consultant frames, an Agentforce agent executes in direct contact with the client stakeholder, and a human checkpoint validates before handing the output to the next phase. Whether this is one continuous agent or several specialized agents orchestrated across phases is a solution-design decision, not assumed here.

Discovery — qualifying a market's readiness

An Agentforce agent conducts the interview end to end with the market's business personas; the consultant only steps in as an interpretation safeguard (human in the loop).

⚙️ Administrative — consultant back-office
1
Questionnaire & country scope set up
Consultant Agent Stakeholder
  • 🧭Selects the market and activates the CROSS / CS / FEX tracks
  • 📝Configures the question set applicable to the country scope
  • 👥Designates the business personas to interview
Output — Questionnaire configured + country scope defined
→
⚙️ Administrative — consultant back-office
2
Cloud & documentation set up
Consultant Agent Stakeholder
  • ☁️Sets up the Cloud environment (CG Cloud / Service Cloud) tied to the market
  • 📚Uploads the reference documentation (process, core model, country standards) the agent will use
Output — Environment + documentation base ready for the agent
→
💬 Client portal — customer facing
3
Questionnaire response (portal)
Consultant Agent Stakeholder
  • Asks the questions in natural language to each persona on the portal
  • Leans on the uploaded documentation to clarify/rephrase a question
  • Follows up and probes incomplete answers
Output — Answers logged in real time in the repository
↩
💬 Client portal — customer facing
4
Signal detection
Consultant Agent Stakeholder
  • Detects sensitive answers (critical flag)
  • Categorizes by dimension — Technical/Process/Gov/Change
Output — Flagged answers ready for scoring
↩
5
Automated scoring
Consultant Agent Stakeholder
  • Converts each answer into a weighted score
  • Aggregates by dimension (35/30/20/15%)
Output — Raw scorecard by dimension
→
6
Validation — Human in the loop
Consultant Agent Stakeholder
  • 🧭Rules on critical flags and ambiguities
  • ✅Validates or corrects the score before it's released
Output — Validated scorecard
→
7
Programme decision
Consultant Agent Stakeholder
  • 📊Presents the scorecard to the steering committee
  • 🎯Decides on deployment, adaptation, or waiting
Output — GO / ADAPT / WAIT verdict
📤 Phase output → Maturity score + GO/ADAPT/WAIT verdict, and the list of focus areas to deep dive in F2T

F2T — Fit to Template

Three things happen at once: the project defines its top-down approach and how much flexibility/localization the platform allows; the client stakeholders self-learn the Core Model with an Agentforce agent's help and deep dive on the Discovery focus areas; and the agent helps surface the real gaps between the local as-is and the Core Model.

⚙️ Project framing — architect / consultant
1
Top-down approach & flexibility level
Consultant Agent Stakeholder
  • 🧭Defines the project's top-down approach to the platform
  • 📐Sets the allowed level of flexibility / local adaptation vs. the standard
Output — Top-down approach + flexibility/localization level defined
→
⚙️ Project framing — architect / consultant
2
Core Model & learning modules set up
Consultant Agent Stakeholder
  • 🏗️The architect defines the Core Model scope modules
  • 📚Defines the self-learning modules the agent will use with stakeholders
Output — Core Model scope + learning modules configured
→
🎓 Self-learning — customer stakeholders
3
Core Model self-learning (portal)
Consultant Agent Stakeholder
  • Familiarizes stakeholders with the Core Model, self-learning style
  • Answers control-check questions grounded in the Core Model's content & documentation
Output — Stakeholders familiar with the Core Model
↩
🎓 Self-learning — customer stakeholders
4
Focus area deep dive
Consultant Agent Stakeholder
  • Returns to the Discovery focus areas with the same personas
  • Digs deeper to learn more and surface the real gaps vs. the Core Model
Output — Focus areas deep-dived, real gaps surfaced
↩
🔍 Gap analysis — as-is vs. Core Model
5
As-is vs. Core Model pattern
Consultant Agent Stakeholder
  • Helps understand each gap and how it differs from the Core Model
  • Lays out the local as-is situation side by side with the Core Model pattern
Output — As-is vs. Core Model gap map per focus area
→
6
Consultant review
Consultant Agent Stakeholder
  • 🧭Validates the qualification of each gap
  • ✅Prioritizes the gaps that will move into Design
Output — Validated and prioritized fit/gap map
→
7
Stakeholder readout
Consultant Agent Stakeholder
  • 📊Shares the gaps with the business stakeholders
  • 🤝Aligns on priorities ahead of Design
Output — Gaps aligned, ready for Design
📤 Phase output → Top-down approach & flexibility level defined, stakeholders familiar with the Core Model, and the detailed as-is vs. Core Model gap map per focus area

Design — framing the gaps and templatizing client deliverables

An Agentforce agent consolidates the gaps surfaced by F2T and generates the templatized core model files (audit activities, email-to-case, etc.) that the consultant and the stakeholder validate together.

1
Gap consolidation
Consultant Agent Stakeholder
  • Aggregates the gaps from F2T by theme
  • Structures a high-level view of the potential gaps
Output — High-level gap report
→
2
Impact analysis
Consultant Agent Stakeholder
  • 🧭Prioritizes critical vs. minor gaps
  • 📐Decides which gaps require a dedicated template
Output — Prioritized gap list to address
→
3
Template generation
Consultant Agent Stakeholder
  • Generates the core model files (audits, email-to-case…)
  • Adapts each template to the market's specifics
Output — First version of the templatized core model files
↩
4
Stakeholder review
Consultant Agent Stakeholder
  • 🔎Checks that the templates reflect the local business
  • 💬Flags the needed adjustments
Output — Consolidated business feedback
→
5
Deliverable adjustment
Consultant Agent Stakeholder
  • Incorporates the feedback into the templatized files
  • Re-validates consistency with the core model
Output — Adjusted templates
→
6
Design sign-off
Consultant Agent Stakeholder
  • ✅Formally validates gaps + templates with the stakeholder
  • 📦Hands the package over to Build
Output — Validated Design package
📤 Phase output → High-level gaps + templatized core model files (audit activities, email-to-case, etc.)

Build — measuring deployment and replication confidence

An Agentforce agent instruments the pilot deployment, cross-checks results against the Design templates, and returns a quantified confidence level — for this market and for its replication to the next ones.

1
Pilot configuration
Consultant Agent Stakeholder
  • 🛠️Configures the environment from the Design package
  • Assists the setup based on the core model templates
Output — Pilot environment configured
→
2
Compliance testing
Consultant Agent Stakeholder
  • Runs the test scenarios against the expected design
  • Detects configuration gaps
Output — Design compliance report
→
3
Business acceptance (UAT)
Consultant Agent Stakeholder
  • 🧪Tests the real business journeys (audits, cases…)
  • 💬Flags field issues
Output — Consolidated UAT feedback
↩
4
Quality monitoring
Consultant Agent Stakeholder
  • Monitors stability and adoption indicators
  • Aggregates tests + UAT + monitoring into one score
Output — Consolidated quality indicators
→
5
Confidence score
Consultant Agent Stakeholder
  • Calculates the deployment confidence level
  • 🧭The consultant validates the score before it's released
Output — Deployment + replication confidence score
→
6
Replication decision
Consultant Agent Stakeholder
  • 📊Presents the score to the steering committee
  • 🎯Go/no-go to replicate to the next markets
Output — Replication decision + rollout plan
📤 Phase output → Confidence level of the deployment and its replicability to the next markets
