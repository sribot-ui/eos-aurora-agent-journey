L'agent EOS Aurora — un opérateur unique sur les 4 phases du programme

De la qualification d'un marché jusqu'à la confiance dans son déploiement, le même agent accompagne consultants Salesforce et stakeholders client à chaque phase — avec un point de contrôle humain à chaque bascule critique.

Executive summary
4 phases, les jalons clés, qui fait quoi

Le même agent accompagne le programme de bout en bout. À chaque phase, le consultant SF cadre et valide, l'agent exécute au contact du stakeholder client — avec un point de contrôle humain avant de transmettre l'output à la phase suivante.

Discovery — qualifier la readiness d'un marché

L'agent conduit l'entretien de bout en bout avec les personas métier du marché ; le consultant n'intervient qu'en garde-fou d'interprétation (human in the loop).

⚙️ Administratif — back-office consultant
1
Set up questionnaire & scope pays
Consultant Agent Stakeholder
  • 🧭Sélectionne le marché et active les volets CROSS / CS / FEX
  • 📝Configure le set de questions applicable au scope pays
  • 👥Désigne les personas métier à interroger
Output — Questionnaire configuré + scope pays défini
→
⚙️ Administratif — back-office consultant
2
Set up cloud & documentation
Consultant Agent Stakeholder
  • ☁️Paramètre l'environnement Cloud (CG Cloud / Service Cloud) associé au marché
  • 📚Dépose la documentation de référence (process, core model, standards pays) que l'agent utilisera
Output — Environnement + base documentaire prêts pour l'agent
→
💬 Portail client — customer facing
3
Réponse au questionnaire (portail)
Consultant Agent Stakeholder
  • Pose les questions en langage naturel à chaque persona sur le portail
  • S'appuie sur la documentation déposée pour éclairer/reformuler une question
  • Relance et approfondit les réponses incomplètes
Output — Réponses consignées en temps réel dans le référentiel
↩
💬 Portail client — customer facing
4
Détection des signaux
Consultant Agent Stakeholder
  • Détecte les réponses sensibles (flag critique)
  • Catégorise par dimension Technical/Process/Gov/Change
Output — Réponses flaggées prêtes pour scoring
↩
5
Scoring automatique
Consultant Agent Stakeholder
  • Convertit chaque réponse en score pondéré
  • Agrège par dimension (35/30/20/15%)
Output — Scorecard brute par dimension
→
6
Validation — Human in the loop
Consultant Agent Stakeholder
  • 🧭Tranche sur les flags critiques et ambiguïtés
  • ✅Valide ou corrige le score avant diffusion
Output — Scorecard validée
→
7
Décision programme
Consultant Agent Stakeholder
  • 📊Présente la scorecard au comité de pilotage
  • 🎯Décide déploiement, adaptation ou attente
Output — Verdict GO / ADAPT / WAIT
📤 Sortie de phase → Score de maturité + verdict GO/ADAPT/WAIT, et la liste des focus areas à approfondir en F2T
2

F2T — Fit to Template

Sur les focus areas dégagées en Discovery, l'agent revient vers les mêmes personas pour un deep dive, et compare leurs pratiques au core model — dans un périmètre d'apprentissage cadré par l'admin (self-learning scope).

1
Sélection des focus areas
Consultant Agent Stakeholder
  • 🧭Reprend les zones à risque identifiées en Discovery
  • 📐Définit le scope self-learning confié à l'agent (admin)
Output — Périmètre de deep dive + scope d'apprentissage cadré
→
2
Deep dive terrain
Consultant Agent Stakeholder
  • Revient vers les mêmes personas que le Discovery
  • Questionne en profondeur sur chaque focus area
Output — Pratiques détaillées documentées par focus area
→
3
Apprentissage continu
Consultant Agent Stakeholder
  • Affine sa compréhension du marché à chaque échange (self-learning)
  • Reste borné au scope défini par l'admin
Output — Modèle de compréhension du marché à jour
↩
4
Comparaison au core model
Consultant Agent Stakeholder
  • Compare chaque pratique locale au core model de référence
  • Qualifie l'écart : fit direct, adaptable, ou gap
Output — Cartographie fit/gap par focus area
→
5
Revue consultant
Consultant Agent Stakeholder
  • 🧭Valide la qualification de chaque écart
  • ✅Priorise les gaps qui iront en Design
Output — Cartographie fit/gap validée et priorisée
→
6
Restitution stakeholders
Consultant Agent Stakeholder
  • 📊Partage les écarts avec les stakeholders métier
  • 🤝Aligne sur les priorités avant le Design
Output — Écarts alignés, prêts pour le Design
📤 Sortie de phase → Cartographie fit/gap détaillée par focus area comparée au core model
3

Design — cadrer les gaps et templatiser les livrables client

L'agent consolide les écarts remontés du F2T et génère les fichiers core model templatisés (activités d'audit, email-to-case, etc.) que le consultant et le stakeholder valident ensemble.

1
Consolidation des gaps
Consultant Agent Stakeholder
  • Agrège les écarts issus du F2T par thème
  • Structure une vue high-level des gaps potentiels
Output — Rapport de gaps high-level
→
2
Analyse d'impact
Consultant Agent Stakeholder
  • 🧭Priorise les gaps critiques vs. mineurs
  • 📐Décide quels gaps nécessitent un template dédié
Output — Liste de gaps priorisés à traiter
→
3
Génération des templates
Consultant Agent Stakeholder
  • Génère les fichiers core model (audits, email-to-case…)
  • Adapte chaque template aux spécificités du marché
Output — 1re version des fichiers core model templatisés
↩
4
Revue stakeholder
Consultant Agent Stakeholder
  • 🔎Vérifie que les templates reflètent le métier local
  • 💬Remonte les ajustements nécessaires
Output — Feedback métier consolidé
→
5
Ajustement des livrables
Consultant Agent Stakeholder
  • Intègre le feedback dans les fichiers templatisés
  • Revalide la cohérence avec le core model
Output — Templates ajustés
→
6
Sign-off Design
Consultant Agent Stakeholder
  • ✅Valide formellement gaps + templates avec le stakeholder
  • 📦Transmet le dossier au Build
Output — Dossier de Design validé
📤 Sortie de phase → Gaps high-level + fichiers core model templatisés (activités d'audit, email-to-case, etc.)
4

Build — mesurer la confiance de déploiement et de réplication

L'agent instrumente le déploiement pilote, croise les résultats aux templates du Design, et restitue un niveau de confiance chiffré — pour ce marché et pour sa réplication aux marchés suivants.

1
Configuration pilote
Consultant Agent Stakeholder
  • 🛠️Configure l'environnement à partir du dossier Design
  • Assiste le paramétrage à partir des templates core model
Output — Environnement pilote configuré
→
2
Tests de conformité
Consultant Agent Stakeholder
  • Exécute les scénarios de test vs. le design attendu
  • Détecte les écarts de configuration
Output — Rapport de conformité au design
→
3
Recette métier (UAT)
Consultant Agent Stakeholder
  • 🧪Teste les parcours métier réels (audits, cases…)
  • 💬Remonte les anomalies terrain
Output — Feedback UAT consolidé
↩
4
Monitoring qualité
Consultant Agent Stakeholder
  • Surveille les indicateurs de stabilité et d'adoption
  • Agrège tests + UAT + monitoring en un score
Output — Indicateurs de qualité consolidés
→
5
Score de confiance
Consultant Agent Stakeholder
  • Calcule le niveau de confiance déploiement
  • 🧭Le consultant valide le score avant diffusion
Output — Score de confiance déploiement + réplication
→
6
Décision de réplication
Consultant Agent Stakeholder
  • 📊Présente le score au comité de pilotage
  • 🎯Go/no-go pour répliquer sur les marchés suivants
Output — Décision de réplication + plan de rollout
📤 Sortie de phase → Niveau de confiance du déploiement et de sa réplicabilité aux marchés suivants
Discovery — zoom
Un questionnaire organisé en 3 volets métier
🧾

CROSS — 13 questions

Comptes clients, prospects, données, conformité GDPR. Le socle commun à tous les marchés.

🎧

CS — 16 questions

Organisation du Service Client, canaux de contact, outils actuels, SLAs.

🚗

FEX — 20 questions

Vente terrain, planification de visites, audits magasin, outils des commerciaux.

⚠️

Flags critiques

Certaines réponses déclenchent une alerte : c'est le signal qui appelle la relecture du consultant, pas une décision automatique.

Pourquoi c'est utile
Ce que la solution apporte concrètement

Objectivité

Remplace une évaluation qualitative subjective par un score reproductible, basé sur des critères documentés et justifiés.

→

Vitesse

Un entretien structuré remplace des semaines d'ateliers exploratoires pour qualifier la readiness d'un marché.

→

Priorisation

Le programme sait exactement quels marchés déployer en premier, et quels risques traiter avant le go-live.

→

Transparence

Chaque score est justifiable auprès des stakeholders locaux — la méthodologie est documentée et explicable.
